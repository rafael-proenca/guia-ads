Introdução à Engenharia de Software

01 - Introdução à Engenharia de Software

A Engenharia de Software é a disciplina que aplica princípios de engenharia para desenvolver, operar e manter software de alta qualidade de forma sistemática e profissional.

  • Importância do Software: O software é onipresente, desde sistemas embarcados em eletrodomésticos até infraestruturas críticas em bancos e hospitais.
  • A Crise do Software: Termo dos anos 1970 para descrever um período em que os projetos de software falhavam consistentemente em cumprir prazos e orçamentos, resultando em produtos de baixa qualidade. Isso impulsionou a necessidade de uma abordagem de engenharia.
  • Stakeholders: São todos os envolvidos e interessados no projeto (gerentes, clientes, usuários, equipe de desenvolvimento). A participação deles é crucial.
  • Causas de Falha em Projetos: Falhas comuns incluem planejamento inadequado, prazos irreais, má gestão de requisitos, pouca participação dos stakeholders e escolha errada da tecnologia ou do processo de desenvolvimento.

02 - Atividades do Processo de Software

Qualquer processo de desenvolvimento de software é composto por um conjunto de atividades fundamentais. A forma como são organizadas define o modelo de ciclo de vida.

  • Especificação (Engenharia de Requisitos): Define o que o sistema deve fazer. É uma fase crítica que envolve o estudo de viabilidade, levantamento (elicitação), análise, especificação e validação dos requisitos.
  • Desenvolvimento (Projeto e Implementação): Traduz os requisitos em um sistema funcional. Envolve o projeto da arquitetura, das interfaces, dos componentes e do banco de dados, seguido pela codificação.
  • Validação (Verificação e Validação - V&V): Garante que o software atende às especificações e às necessidades do cliente. A principal ferramenta são os testes, que ocorrem em diferentes níveis (componente, sistema e aceitação).
  • Evolução (Manutenção): Garante que o software continue útil e atualizado após a entrega, adaptando-se a novas necessidades e corrigindo defeitos. É, frequentemente, a fase mais longa e custosa do ciclo de vida.

03 - Ciclo de Vida e Modelos Sequenciais

O ciclo de vida do software define as etapas a serem seguidas durante a construção do produto. Os modelos organizam essas etapas.

  • Modelo em Cascata (Waterfall): É o modelo mais antigo e estritamente sequencial. Uma fase só começa após a anterior terminar completamente (Requisitos → Projeto → Implementação → Teste → Manutenção).
    • Vantagens: Simples de entender e gerenciar.
    • Desvantagens: Extremamente rígido, não lida bem com mudanças de requisitos e o cliente só vê o produto no final. Ideal apenas para projetos pequenos com requisitos muito bem definidos e estáveis.

04 - Modelos Iterativos e Evolucionários

Esses modelos surgiram para dar mais flexibilidade ao desenvolvimento, permitindo mudanças e entregas mais rápidas de valor.

  • Desenvolvimento Incremental: O software é desenvolvido e entregue em partes (incrementos). A cada incremento, uma nova funcionalidade é adicionada ao sistema.
  • Desenvolvimento Iterativo: O software é desenvolvido em ciclos (iterações). A cada ciclo, o sistema como um todo é refinado e melhorado.
  • Modelo de Prototipação: Um modelo iterativo focado na construção rápida de protótipos (versões simplificadas) para obter feedback do usuário, ajudando a esclarecer requisitos e validar o design da interface.
  • Modelo Espiral: Um modelo evolucionário sofisticado que combina iteração com o controle do modelo em cascata. Seu diferencial é ser guiado por análise de riscos. Cada ciclo da espiral passa por quatro fases: planejamento, análise de risco, engenharia e avaliação do cliente.

05 - Modelo de Processo Unificado (UP)

O Processo Unificado (UP) é um framework de desenvolvimento moderno, híbrido e adaptável, que consolida as melhores práticas dos modelos anteriores.

  • Principais Características:
    • É Iterativo e Incremental.
    • É Orientado a Casos de Uso para capturar requisitos.
    • É Centrado na Arquitetura, definindo uma base sólida no início.
    • É Focado em Riscos, buscando mitigá-los o mais cedo possível.
  • Fases do UP: O projeto é dividido em quatro fases sequenciais.
    1. Concepção: Define a visão geral, o escopo e a viabilidade do projeto.
    2. Elaboração: Detalha os requisitos, estabelece a arquitetura e trata os principais riscos.
    3. Construção: Desenvolve o software de forma incremental até que esteja completo.
    4. Transição: Entrega o software aos usuários (implantação, treinamento).
  • Implementações: O UP é um framework, não uma metodologia rígida. Suas principais implementações são o Rational Unified Process (RUP), que é mais robusto e prescritivo, e o OpenUP, que é uma versão mais leve e ágil.

Conclusão: Pontos Principais

  • Evolução de Processos: A Engenharia de Software evoluiu de processos rígidos e sequenciais (Cascata) para modelos flexíveis, iterativos e incrementais (Espiral, UP).
  • Gerenciamento de Riscos: A capacidade de identificar e mitigar riscos no início do projeto (uma característica central do Espiral e do UP) é fundamental para o sucesso.
  • Foco no Cliente e no Valor: Modelos modernos priorizam a colaboração com o cliente e a entrega contínua de funcionalidades úteis, em vez de esperar até o final do projeto.
  • Atividades Fundamentais: Independentemente do modelo, as atividades de especificação, desenvolvimento, validação e evolução são sempre necessárias.
  • Não existe "bala de prata": A escolha do modelo de ciclo de vida correto depende da natureza do projeto (tamanho, complexidade, estabilidade dos requisitos).
