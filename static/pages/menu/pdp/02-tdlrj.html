Tomando Decisão e Laços de Repetição em JavaScript

01 - Estrutura Condicional if/else e Operador Ternário

Estruturas condicionais permitem que um programa execute diferentes blocos de código com base em condições que podem ser verdadeiras ou falsas.

  • if: Executa um bloco de código se a condição for verdadeira.
  • if...else: Executa um bloco de código se a condição for verdadeira e outro bloco se for falsa.
  • if...else if...else: Testa múltiplas condições em sequência. É útil para cenários com várias possibilidades.
  • Operador Ternário (? :): É uma forma compacta de escrever uma declaração if...else em uma única linha, melhorando a legibilidade para condições simples.

Exemplo de if...else if...else:


let idade = 25;

if (idade < 16) {
    console.log("Não eleitor");
} else if (idade >= 18 && idade < 65) {
    console.log("Eleitor Obrigatório");
} else {
    console.log("Eleitor Facultativo");
}

Exemplo de Operador Ternário:


let idade = 18;
idade<18 ? console.log("Menor de idade") : console.log("Maior de idade");
//ou
let idade = 18;
idade<18 ? console.log("Menor de idade") : idade<60 ? console.log("Adulto") : console.log("Idoso");

02 - Estrutura Condicional Switch Case

O switch é uma alternativa ao if...else if...else, ideal para quando se precisa testar uma única variável contra múltiplos valores possíveis.

  • case: Define um dos valores a serem comparados com a variável.
  • break: Encerra a execução do switch. Se omitido, o código continuará a executar os próximos cases.
  • default: Bloco de código a ser executado se nenhum dos cases corresponder ao valor da variável.

Exemplo de switch case:


let mes = 8;
let nomeMes;

switch (mes) {
  case 1:
    nomeMes = "Janeiro";
    break;
  case 8:
    nomeMes = "Agosto";
    break;
  default:
    nomeMes = "Mês inexistente";
}
console.log("O mês escolhido é: " + nomeMes); // Saída: O mês escolhido é: Agosto

03 - Laços de Repetição While e Do...While

Laços de repetição (ou loops) executam um bloco de código repetidamente enquanto uma condição for verdadeira.

  • while: Testa a condição antes de cada execução do bloco. Se a condição for falsa desde o início, o bloco nunca é executado.
  • do...while: Testa a condição depois de cada execução do bloco. Isso garante que o bloco de código seja executado pelo menos uma vez.

Exemplo de while:


// Exemplo com o laço 'while'
console.log("--- Exemplo com while ---");
let i = 5;

while (i < 5) {
  console.log("O valor de i é: " + i);
  i++;
}

console.log("Loop 'while' finalizado. Valor final de i: " + i);


// Exemplo com o laço 'do-while'
console.log("\n--- Exemplo com do-while ---");
let j = 5;

do {
  console.log("O valor de j é: " + j);
  j++;
} while (j < 5);

console.log("Loop 'do-while' finalizado. Valor final de j: " + j);

04 - Laços de Repetição for, for...in e for...of

A estrutura for e suas variações são as mais utilizadas para criar laços de repetição, por serem mais compactas e organizadas.

  • for: É o laço mais tradicional. A inicialização da variável, o teste da condição e o incremento/decremento são definidos em uma única linha.
  • for...in: Percorre as propriedades (chaves) de um objeto.
  • for...of: Percorre os valores de objetos iteráveis, como Arrays e Strings.
  • Tanto no for...in quanto no for...of, um novo "escopo" é criado para a variável a cada iteração do loop. Portanto, const é a escolha ideal, a menos que a variável precise sofrer alguma alteração no interior do loop. É uma boa prática de programação dar às suas variáveis a menor permissão possível. Se ela não precisa mudar, ela não deve poder mudar (Princípio do Menor Privilégio).

Exemplos de laços for:

for (tradicional):


console.log("Contagem Regressiva:")
for (let i = 5; i>= 0; i--){
  console.log(i);
}
console.log("DECOLAR!!!");

for...in (percorrendo objeto):


const usuario = {
  //propriedades
  nome: "user",
  idade: "18",
  email:"abcd@wxyz.com",
  //métodos  
  apresentacao: function () {
    return `Olá, meu nome é ${this.nome}, tenho ${this.idade} anos e meu email é ${this.email}.`;
  }
}
//acesso estático
console.log(usuario.nome);
console.log(usuario.apresentacao());
//acesso dinâmico
for (const propriedade in usuario){
  console.log(`${propriedade} : ${usuario[propriedade]}`);
}
//arrays
const cars = ["Saab", "Volvo", "BMW"];
for (const car in cars) {
  console.log(`Carro ${car}: ${cars[car]}`);
}

for...of (percorrendo array):


//arrays
const cars = ["Saab", "Volvo", "BMW"];
for (const car of cars) {
  console.log(`Carro: ${car}`);
}

Conclusão: Pontos Principais

  • Estruturas de Decisão: O if/else é flexível e ideal para testar múltiplas condições e intervalos. O switch é mais limpo e organizado para testar uma única variável contra valores específicos.
  • Estruturas de Repetição: O while testa a condição no início. O do...while testa no final, garantindo ao menos uma execução.
  • O poder do for: O laço for é a estrutura de repetição mais comum e versátil. Use for...in para propriedades de objetos e for...of para valores de iteráveis como arrays.
  • Escolha Certa: A escolha da estrutura correta (condicional ou de repetição) é fundamental para criar um código claro, eficiente e sem erros.
